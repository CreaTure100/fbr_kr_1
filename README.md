## 📋 Описание практических работ

### Практики 19–21: Основы веб-разработки и создание API
В рамках этих работ была реализована базовая клиент-серверная архитектура приложения:
- Верстка пользовательского интерфейса с использованием **HTML, CSS и SCSS**.
- Написание клиентской логики на **JavaScript**.
- Разработка серверной части (Backend) на базе **Node.js и Express**, обработка HTTP-запросов и создание REST API.

### Практика 22: Балансировка нагрузки (Load Balancing)
Изучение подходов к распределению входящего трафика для повышения отказоустойчивости и производительности:
- Запуск нескольких экземпляров backend-сервера на разных портах (3001, 3002, 3003).
- Настройка программного балансировщика **Nginx** с использованием алгоритма *Round Robin*.
- Реализация отказоустойчивости с помощью параметров `max_fails` и `fail_timeout` (автоматическое переключение на резервный `backup` сервер при падении основных).
- Альтернативная реализация балансировки с использованием **HAProxy** и настроек health checks.

### Практика 23: Контейнеризация с Docker
Перенос микросервисной архитектуры в изолированные контейнеры:
- Написание `Dockerfile` для создания легковесного образа Node.js приложения (на базе `alpine`).
- Написание `docker-compose.yml` для автоматического развертывания многоконтейнерного приложения.
- Настройка единой внутренней сети (`bridge`) для безопасного общения контейнеров.
- Связка Nginx с изолированными backend-контейнерами (обращение по именам сервисов, а не по IP).

## 🚀 Как запустить проект (на примере Практики 23)

Для запуска финальной микросервисной архитектуры вам потребуется установленный **Docker** и **Docker Compose**.

1. **Склонируйте репозиторий:**
   ```bash
   git clone https://github.com/CreaTure100/fbr_kr_1.git
   cd fbr_kr_1
   ```

2. **Перейдите в директорию с 23-й практикой:**
   ```bash
   cd "practice 23"
   ```

3. **Запустите сборку и старт контейнеров:**
   ```bash
   docker compose up --build
   ```

4. **Проверьте работу приложения:**
   Откройте браузер или используйте `curl`:
   ```bash
   curl http://localhost/
   ```
   *При каждом обновлении страницы вы будете видеть ответы от разных backend-контейнеров, что подтверждает успешную работу балансировщика Nginx внутри Docker-сети.*

5. **Остановка контейнеров:**
   Чтобы остановить работу, нажмите `Ctrl + C` в терминале или выполните команду:
   ```bash
   docker compose down --remove-orphans
   ```
